# 모달 폼 사용성 개선 설계

## 목표

새 조직, 새 프로젝트, 프로젝트 수정 모달에서 입력 필드와 실행 버튼의
위계를 명확하게 만들고 버튼 사이에 일관된 간격을 둔다. 프로젝트 날짜는
달력 아이콘뿐 아니라 날짜 입력 영역 전체를 클릭해 네이티브 달력을 열 수
있게 한다.

## 범위

- 새 조직 모달
- 새 프로젝트 모달
- 프로젝트 수정 및 일정 수정 모달
- 공통 모달 액션 행
- 공통 날짜 입력 컴포넌트

프로젝트 상세 화면, 일반 텍스트 입력, 커스텀 달력 UI, 날짜 저장 형식과
API 계약은 변경하지 않는다.

## 선택한 접근

날짜 입력 전용 `DateInput` 컴포넌트와 모달 폼 전용 레이아웃 class를
추가한다. 일반 `TextInput`에 날짜 전용 동작을 섞지 않고 생성·수정 모달이
동일한 날짜 상호작용을 재사용하게 한다.

브라우저가 `HTMLInputElement.showPicker()`를 지원하면 날짜 입력 영역을
클릭했을 때 이를 호출한다. 지원하지 않거나 호출이 허용되지 않는
브라우저에서는 입력에 focus를 주고 기존 네이티브 `type="date"` 동작을
유지한다. 별도 날짜 라이브러리나 커스텀 달력은 추가하지 않는다.

## 레이아웃

### 프로젝트 생성·수정

- 프로젝트 이름은 첫 번째 행 전체 너비를 사용한다.
- 시작일과 종료일은 두 번째 행에서 같은 너비로 나란히 배치한다.
- 종료일이 시작일보다 빠른 오류는 날짜 행 아래 전체 너비에 표시한다.
- `닫기`와 `프로젝트 만들기` 또는 `저장`은 폼 아래의 공통 액션 행에
  배치한다.
- 액션 행은 데스크톱에서 오른쪽 정렬하고 버튼 사이 간격은
  `var(--er-space-3)`으로 한다.
- 36rem 이하 화면에서는 날짜 필드를 한 열로 쌓고 액션 버튼은 사용할 수
  있는 너비 안에서 명확한 간격을 유지한다.

### 조직 생성

- 조직 이름 입력은 한 행 전체 너비를 사용한다.
- `닫기`와 `조직 만들기`는 입력 아래 공통 액션 행에 나란히 배치한다.
- 프로젝트 모달과 같은 버튼 순서, 정렬, 간격을 사용한다.

## 컴포넌트 책임

### `DateInput`

- label, value, change event, disabled 등 네이티브 날짜 입력 속성을 받는다.
- label과 input의 접근 가능한 연결을 유지한다.
- 입력 자체를 클릭했을 때 지원 브라우저에서는 `showPicker()`를 호출한다.
- label 클릭은 표준 `htmlFor` 연결을 통해 input click으로 전달되며 별도
  가짜 click target을 만들지 않는다.
- 키보드 focus, Tab 순서, 브라우저 기본 날짜 입력 조작을 방해하지 않는다.
- `showPicker()`가 없거나 `InvalidStateError`, `NotAllowedError` 등으로
  실패하면 예외를 외부로 전파하지 않고 focus된 네이티브 입력을 유지한다.

### `Dialog`

기존 focus trap, Escape 닫기, 열기 전 요소로 focus 복원 책임을 유지한다.
기본 `닫기` 버튼을 선택적으로 숨길 수 있는
`hideDefaultCloseAction?: boolean` prop을 추가한다. 기본값은 `false`로
기존 dialog의 children 뒤에 현재 `닫기` 버튼을 그대로 제공한다.

새 조직, 새 프로젝트, 프로젝트 수정 모달은 이 prop을 `true`로 설정하고
각 `<form>` 내부의 `er-dialog-actions`에 `닫기`와 submit 버튼을 직접
렌더링한다. 이렇게 하면 submit 버튼과 form 연결을 위한 별도 ID나
`form` 속성이 필요 없고, 나머지 dialog 호출부도 변경하지 않는다.

### 모달 폼 class

- `er-dialog-form`: 한 열 폼 컨테이너와 수직 간격
- `er-dialog-form__dates`: 두 날짜 필드의 반응형 grid
- `er-dialog-form__error`: 전체 너비 오류 영역
- `er-dialog-actions`: 오른쪽 정렬된 버튼 행과
  `var(--er-space-3)` 간격

이 class들은 일반 목록 필터에 사용하는 `er-form-grid`를 변경하지 않는다.

## 상호작용과 오류 처리

- 시작일과 종료일은 계속 선택 사항이다.
- 두 날짜가 모두 있고 종료일이 시작일보다 빠르면 기존 오류 문구와 제출
  차단을 유지한다.
- 조직 이름이나 프로젝트 이름이 비어 있으면 기존처럼 제출 버튼이
  비활성화된다.
- 저장 중 버튼 비활성화 동작을 유지한다.
- 날짜 picker 호출 실패는 사용자 입력 실패가 아니므로 별도 오류 문구를
  표시하지 않는다.
- 닫기, Escape, backdrop 상태, 제출 성공·실패 데이터 흐름은 변경하지
  않는다.

## 접근성

- 날짜 입력은 실제 `input type="date"`를 유지한다.
- label 클릭과 input 클릭 모두 동일한 입력으로 연결된다.
- 날짜 영역은 별도 가짜 button 역할을 만들지 않아 중복 focus target을
  만들지 않는다.
- 액션 순서는 DOM에서 `닫기`가 먼저, 주요 실행 버튼이 다음이다.
- focus trap은 액션 행의 두 버튼을 포함하고 Escape 닫기와 focus 복원을
  유지한다.
- disabled 날짜 입력에서는 picker를 열지 않는다.

## 테스트

- `DateInput` 클릭 시 지원 환경에서 `showPicker()`가 호출된다.
- label 클릭도 표준 label/input 연결을 통해 input focus와 picker 호출로
  연결된다.
- `showPicker()` 미지원 또는 예외 발생 시 컴포넌트가 깨지지 않는다.
- disabled 입력은 picker를 호출하지 않는다.
- 새 프로젝트와 프로젝트 수정 모달이 이름 행, 날짜 행, 오류 영역, 액션
  행 class를 사용한다.
- 새 조직 모달에 `닫기`, `조직 만들기`가 같은 액션 행과 올바른 DOM 순서로
  존재한다.
- 기존 날짜 역전 validation과 API payload 테스트를 유지한다.
- Dialog의 Tab/Shift+Tab focus trap, Escape 닫기, focus 복원 테스트를
  유지한다.
- production build와 전체 Web 테스트에서 회귀가 없어야 한다.

## 완료 기준

- 세 모달의 버튼이 하단 액션 행에 나란히 있고 버튼 사이에 일관된 간격이
  보인다.
- 프로젝트 이름은 날짜와 섞이지 않고 첫 행 전체 너비를 사용한다.
- 시작일과 종료일은 데스크톱에서 같은 행, 모바일에서 세로로 표시된다.
- 날짜 입력의 보이는 영역을 클릭하면 가능한 브라우저에서 네이티브 달력이
  열린다.
- 키보드 조작과 기존 validation, 제출, 닫기 동작이 유지된다.
